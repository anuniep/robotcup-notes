# 🤖 日常如何写代码和笔记上传到 GitHub

在 Windows 的 `D:\Code\note` 里写了新笔记（如 `ros2_notes.md`），或者在 `D:\Code` 下修改了机器人代码，想要同步到 GitHub 时，直接复制并运行以下流程即可。

---

## 🔄 第一部分：日常更新同步

请打开 **WSL2 终端（小黑窗）**，依次执行以下命令：

### 1. 进入本地仓库文件夹

```bash
cd ~/robotcup-notes
```  

---

### 2. 将 Windows 路径下的成果复制到当前 WSL2 仓库

**情况 A**：如果只想同步单篇新笔记（注意命令末尾有 空格+点斜杠 ./）

```bash
cp /mnt/d/Code/note/ros2_notes.md ./
```

**情况 B**：如果想一次性把 Windows 该文件夹下所有的 .md 笔记全部同步过来  

```bash
cp /mnt/d/Code/note/*.md ./
```

**情况 C**：如果想同步你在 Windows 写好的机器人源代码

```bash  
cp /mnt/d/Code/your_robot_code.cpp ./
```

---

### 3.git上传步骤

文件复制完成后，熟练地敲下这三行命令：

第一：暂存修改（告诉 Git 准备提交当前目录下的所有新变化）

```bash
git add .
```

第二：记录本地历史（为这次提交打上标签，双引号里写你本次修改的备注）

```bash
git commit -m "更新了ROS2集训笔记与核心控制代码"
```

第三：推向云端（通过 SSH 密钥秒同步到 GitHub 网页端，无需输密码）

```bash
git push
```

---

## 👌第二部分：日常实用排查指令

如果心里不踏实，可以用这两个命令看一眼仓库状态：

查看当前仓库里到底有哪些文件：

```bash
ls -lh
```

查看当前的 Git 状态（有哪些文件被修改了、哪些还没提交）：

```bash
git status
```

---

## 🙌第三部分：团队协作与互看指南

此部分展示代码和笔记的共享步骤：

### 1. 别人如何看你的笔记/代码？

· **如果是公开仓库** (Public)：  

直接把你的 GitHub 仓库网页链接发送给别人（例如：<https://github.com/你的用户名/robotcup-notes>）。他们点击就能在网页上直接以精美的排版阅读你的笔记。

· **如果是私有仓库** (Private)：

  1. 首先进入你的 `GitHub 仓库网页`，点击右上角 `Settings`。
  2. 然后在左侧边栏点击 `Collaborators`，点击 `Add people`。
  3. 最后输入队友的 `GitHub 用户名或邮箱发送邀请`，对方接受后即可查看。

### 2. 你如何看别人的笔记/代码？

· **在线阅读**：直接在 GitHub 网页端搜索并点进他人的仓库，点击任何 .md 文件即可直接阅读。  
· **拉取到本地研究**：点击对方仓库的绿色 Code 按钮，复制 SSH 或 HTTPS 地址，在你的 WSL2 终端运行：

  ```bash
  git clone 对方的仓库地址
  ```

  克隆成功后，使用 cd 对方仓库名 进入，再输入 code . 即可在 VS Code 里一行行研究和调试！
